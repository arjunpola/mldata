{% trans "Our HDF5 Format Explained" %}

HDF5 {% trans "is a data model, library, and file format for storing and managing data. It supports an unlimited variety of datatypes, and is designed for flexible and efficient I/O and for high volume and complex data. HDF5 is portable and is extensible, allowing applications to evolve in their use of HDF5. The HDF5 Technology suite includes tools and applications for managing, manipulating, viewing, and analyzing data in the HDF5 format." %}

{% trans "Internally, we make use of HDF5 because of its hierarchical structure which allows for efficient access of data attributes. It also allows for storing data in a flexible way which seems paramount to us in the current state of the machine learning community and its wild bunch of file formats used." %}

{% trans "The basic abstraction is that a data set is a large collection of objects having the same type, so to say a large array." %}

{% trans "We defined the following HDF5 attributes in our files" %}:

  • mldata: {% trans "Internal mldata version" %}; {% trans "Integer, currently" %} 0
  • name: {% trans "Name to describe the item as found in the file" %}; {% trans "String" %}
  • comment: {% trans "Any comment or description in the file" %}; {% trans "String" %}

{% trans "The following datasets may be defined in our HDF5 data files" %}:

  • attribute_names: {% trans "Names of the data attributes, if available" %}; {% trans "Array of Strings" %}
  • attribute_types: {% trans "Types of the data attributes, if available" %}; {% trans "Array of Strings" %}
  • attributes: {% trans "The data attributes themselves for dense matrices" %}; {% trans "Array of Objects as described by attribute_types" %}
  • attributes_indices: {% trans "The indices of data attributes for sparse matrices" %}; {% trans "Array of Integers" %}
  • attributes_indptr: {% trans "The index pointers of data attributes for sparse matrices" %}; {% trans "Array of Integers" %}
  • attributes_data: {% trans "The data attributes themselves for sparse matrices" %}; {% trans "Array of Objects as described by attribute_types" %}

{% trans "Please note that attributes and attributes_indices + attributes_indptr + attributes_data are mutually exclusive." %}

{% trans "Attribute Types" %}

{% trans "We make a distinction between the actual binary format and the attribute type stored in the file. For example, nominal data might be stored as integers or even bytes for efficiency, although they are mapped to symbolic names to the outside." %}
{% trans "The attribute type is specified by a string which has to be parsed for better flexibility." %}

{% trans "Supported types are:" %}:

  • "NUMERIC"
    Numerical values
    Array of Numerical type
  • "NOMINAL(VALUE1, VALUE2, ...)"
    Enumeration type
    Array of small Integer types
  • "STRING"

    Array of Strings

{% trans "As we see more data types, more type descriptors will be added." %}

{% trans "Split files and Tasks" %}

{% trans "Note that the distinction what is input/output, label/target depends on the TASK, not on the data set itself! We do have a mechanism in place to create automatic split files while slurping datasets from other repositories, though. These datasets may be defined in the split files:" %}

  • labels: {% trans "Index of labels/target for each data row" %}; {% trans "Array of Integers" %}
  • testing/training/validation/whatever the slurper could derive from the dataset: {% trans "Indices of input/output/whatever task requires" %}; {% trans "Array of Integers" %}
