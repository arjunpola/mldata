  4  Structure of the HDF5 Format 
  5  ============================ 
  6   
  7  This will be the generic storage for all kinds of data sets. The basic 
  8  abstraction is that a data set is a large collection of objects having 
  9  the same type, so to say a large array. 
 10   
 11  /mldata           integer = 0 
 12  /name             Name of the data set 
 13  /comment          Initial comment 
 14  /attribute_names  Array of Strings: names of the attributes 
 15  /attribute_types  Array of Strings: description of the attribute types (see below) 
 16  /attributes       Array of Objects as described by attribute_types 
 17   
 18  Note that the distinction what is input/output, label/target depends 
 19  on the TASK, not on the data set itself! 
 20   
 21  Attribute Types 
 22  --------------- 
 23   
 24  We make a distinction between the actual binary format and the 
 25  attribute type stored in the file. For example, nominal data might be 
 26  stored as integers or even bytes for efficiency, although they are 
 27  mapped to symbolic names to the outside. 
 28   
 29  The attribute type is specified by a string which has to be parsed for 
 30  better flexibility. 
 31   
 32  Supported types are: 
 33   
 34  "NUMERIC" 
 35  Numerical values 
 36  Stored as: array of numerical type 
 37   
 38  "NOMINAL(VALUE1, VALUE2, ...)" 
 39  Enumeration type 
 40  Stored as: array of small integer types 
 41   
 42  "STRING" 
 43  Stored as: an array of strings 
 44   
 45  As we see more data types, more type descriptors will be added. 
 46   
 47   
 48  Currently supported formats 
 49  =========================== 
 50  to hdf5 
 51  LibSVM 
 52  ARFF
